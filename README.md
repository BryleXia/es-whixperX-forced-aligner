# 西语字幕强制对齐工具

> 把西班牙语字幕文件的时间戳准确对齐到音频上。在5段纪录片音频（390条字幕，约56分钟）上经过人工逐句审听，**0条错误**。

**作者：** BryleXia · 北京第二外国语学院欧洲学院  
**核心脚本：** `align_srt_routeA.py` · **里程碑标签：** `v13-milestone`

---

## 背景

这个项目的任务是：给一批西班牙语纪录片的字幕文件重新生成时间戳。字幕的文字内容是经过人工校对的，完全可信；需要解决的只是"每句话什么时候说的"这个问题。

听起来简单，但有一个棘手的地方——这批字幕来自中国纪录片的西语版，里面夹着大量中文人名、地名的音译词：

| 正确写法 | AI 语音识别（Whisper）的输出 |
|---------|--------------------------|
| Lao Bao（老包） | "Alien"、"Lab"、"Labao" |
| Nuodeng（诺邓） | "Nudon"、"Nodeng"、"Nuodong" |
| Shengwu（生五） | "Chengwu"、"Shengoo" |
| A Liang（阿亮） | "Alien"（完全无法识别） |

这些词在西班牙语里压根不存在，AI 没见过，只能乱猜。一旦识别出来的文字和字幕对不上，时间戳就跑偏了。

---

## 核心思路

常规做法是：先用 AI 把音频转录成文字，再把转录结果和字幕文字做匹配，最后从匹配结果里提取时间戳。问题出在第一步——专名识别错了，后面全乱。

我们换了一个角度：**既然字幕文字本来就是对的，为什么还要先转录再匹配？直接跳过转录，把字幕文字送进对齐器不就好了？**

```
常规流程：  音频 → [Whisper 转录] → 含错误的文字 → [对齐] → 时间戳
                                          ↑ 专名在此出问题

本方案：    音频 ──────────────────────────────→ [对齐] ← 字幕文字 → 时间戳
```

用这个方法，对齐器（wav2vec2）拿到的是正确的字幕文字，直接在音频里找每个词的位置，不经过 AI 转录这一层。专名识别问题从根本上不再存在——因为我们根本不做转录。

核心代码非常简洁，把所有字幕行合并成一段文本，告诉对齐器"这些词在 0 秒到音频结束这段时间里"，让它自己去找：

```python
full_text = " ".join(srt_lines)
segments  = [{"text": full_text, "start": 0.0, "end": audio_duration}]
aligned   = whisperx.align(segments, align_model, metadata, audio, device)
```

之后再用动态规划（LCS 全局对齐）把词级时间戳映射回每一行字幕。

## 还有一个问题：时间戳偶尔会提前跑

这个方案跑起来之后，发现了一个新问题：在场景切换（长时间静音）的地方，对齐器有时会把下一句字幕的起点放得过早，早了几秒甚至十几秒。

原因是：对齐器必须把每一个词都安排到音频的某一帧上，遇到静音它不会自动跳过，就只能把词往前堆、拉伸时长。结果那句字幕的"词速"就异常低——正常说话2-4个词/秒，被拉伸后可能只有0.5个词/秒。

我们设计了一个检测方法，同时满足以下三个条件时认为这句字幕出了问题：

1. **词速低于 1.3 词/秒**（远低于正常语速）
2. **这句字幕时长超过 6 秒**（排除本身就说得慢的短句）
3. **和前一句字幕的间隔小于 0.3 秒**（正常停顿至少有几百毫秒；间隔极小说明是被对齐器强行塞进去的，不是自然找到的起点）

一旦判定为异常，就用语音活动检测（Silero-VAD）在这句字幕的时间范围内找到真正的语音起点，把时间戳挪过去。

在我们的测试集上，这个方法触发了 3 次，全部修正正确，0 次误判：

| 位置 | 修正前起点 | 修正后起点 | 早了多少 |
|------|----------|----------|---------|
| 第3段第16句 | 02:00 | **02:17** | 17 秒 |
| 第3段第37句 | 04:44 | **04:51** | 7 秒 |
| 第5段第6句 | 00:39 | **00:45** | 6 秒 |

## 实验结果

分两个阶段对比：

| 方案 | 测试句数 | 出错句数 | 错误率 |
|------|---------|---------|--------|
| 旧方案：Whisper 转录 + 模糊匹配 | 237 | 9 | **3.8%** |
| 本方案：跳过转录（无守门员） | 390 | 3 | 0.8% |
| **本方案 v13：跳过转录 + 守门员修正** | **390** | **0** | **0%** |

旧方案里那几个死活修不好的专名（比如"A Liang"被识别为"Alien"，编辑距离太大无法匹配），在新方案里自然消失，不需要任何额外处理。

## 与其他工具的对比

| 工具 | 需要手写发音词典 | 需要 AI 转录 | 单段时长限制 |
|------|:-----------:|:---------:|:----------:|
| MFA（Montreal Forced Aligner） | 是（需手动编写） | 否 | 无 |
| WhisperX 标准流程 | 否 | **是** | 无 |
| Qwen3-ForcedAligner-0.6B（2026） | 否 | 否 | **300 秒** |
| **本方案** | **否** | **否** | **无** |

在精度方面，有研究（Rousso et al., Interspeech 2024）在标准语音测试集（英语）上做过对比：MFA 的词级精度约为 WhisperX 的两倍。本方案绕过了 WhisperX 转录误差这一环节，在我们的场景下（有正确参考文本）达到了 100% 感知准确率。

## 使用方法

### 安装依赖

```bash
pip install whisperx silero-vad num2words imageio-ffmpeg
```

### 准备文件

把音频和字幕放在同一个目录，文件名保持一致（只有扩展名不同）：

```
/root/
├── seg001.m4a
├── seg001.asr.qc.srt   ← 已质检的字幕文本
├── seg002.m4a
├── seg002.asr.qc.srt
└── ...
```

### 运行

```bash
python align_srt_routeA.py
```

对齐后的字幕保存在 `/root/aligned_routeA/` 目录下。

### 修改路径

打开脚本开头修改以下几行：

```python
AUDIO_DIR  = Path("/root")                   # 音频和字幕所在目录
OUTPUT_DIR = Path("/root/aligned_routeA")    # 输出目录
DEVICE     = "cuda"                          # 有 GPU 用 "cuda"，没有用 "cpu"
LANGUAGE   = "es"                            # 语言代码，西语填 "es"
```

### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_words_per_sec` | 1.3 | 低于此词速触发检测 |
| `min_suspect_duration` | 6.0 秒 | 短于此时长的句子不检测 |
| `min_move_s` | 1.5 秒 | 搜索真实起点时向后跳过的最小距离 |

---

## 局限性

- **字幕文字必须正确**：方案的前提是参考文本可信，有错字或漏句时无法自动纠正。
- **超长音频需要分块**：单段超过 30 分钟时自动分块处理，分块边界处可能有轻微误差。
- **仅在西班牙语上完整验证**：理论上支持 whisperx 的所有语言，但其他语种未经测试。
